# 云廊 MVP

这是一个把 115 网盘资料做成私人画廊的第一版原型。当前版本先完成核心浏览体验，并把 115 对接信息集中放在 `115-config.js`。

## 现在能做什么

- 以画廊方式浏览文件。
- 按目录、类型、关键词筛选。
- 识别图片、视频、文档、压缩包等类型。
- 支持收藏和稍后整理。
- 支持导入本地文件夹，用真实图片和视频预览来模拟网盘目录。
- 只改 `115-config.js` 就能切换到 115 数据源。

## 使用方式

推荐启动本地服务：

```bash
node server.js
```

然后打开：

`http://localhost:8787`

也可以直接打开：

`index.html`

打开后可以先看示例数据，也可以点左侧“导入本地文件夹”，选择一个本地目录体验真实画廊效果。

## 后续对接 115 时填哪里

只需要打开：

`115-config.js`

填写这几项：

```js
window.YUNLANG_115_CONFIG = {
  mode: "115",
  apiBaseUrl: "你的 115 API 转接服务地址",
  refreshToken: "你的 115 refresh token",
  rootFolderId: "想作为首页的 115 文件夹 ID，可留空",
  pageSize: 200,
};
```

说明：

- `mode`：从 `"mock"` 改成 `"115"` 后，页面会使用真实数据。
- `apiBaseUrl`：建议填你自己的本地或私有后端地址，由它去调用 115 Open API。
- `refreshToken`：115 授权后得到的长期刷新令牌。
- `rootFolderId`：指定从哪个文件夹开始展示；留空就从根目录开始。
- `pageSize`：每次读取的文件数量。

## 这些信息如何获取

### mode

这个不用获取，手动改：

```js
mode: "115"
```

### apiBaseUrl

这是本项目自带的“115 转接服务”地址，不是 115 官网直接给你的。原因是浏览器页面不适合直接处理 OAuth 刷新流程。

本地开发时通常会是：

```js
apiBaseUrl: "http://localhost:8787"
```

等做成正式桌面应用时，可以把这个转接服务内置进去，用户就不需要看到这个字段。

### refreshToken

这个来自 115 授权流程。常见路径有两种：

1. 在 115 开放平台注册/创建应用，拿到应用信息，然后走官方 OAuth 授权流程获取。
2. 临时验证阶段，可以参考 AList 的 115 Open API 驱动文档获取 refresh token，再填入这里。

拿到后填：

```js
refreshToken: "这里粘贴你的 refresh token"
```

注意：不要把这个文件发给别人。

### rootFolderId

这是你想作为画廊首页的 115 文件夹 ID。可以先留空：

```js
rootFolderId: ""
```

后续接通文件列表后，点开某个目录时就能从接口返回里看到它的 `id`，再复制过来。

### pageSize

这个不用获取，先保持：

```js
pageSize: 200
```

浏览器页面里不建议直接保存长期密钥。第一版为了方便验证，把令牌放在配置文件；正式桌面版建议改成系统钥匙串或本地加密存储。

项目已经提供了这个转接服务：

`server.js`

它会提供一个简单接口：

`GET /files?parentId=root&limit=200`

返回以下格式之一都可以：

```json
{
  "files": [
    {
      "id": "文件 ID",
      "name": "文件名.jpg",
      "path": "照片/文件名.jpg",
      "folder": "照片",
      "size": 123456,
      "updatedAt": "2026-06-06T12:00:00Z",
      "thumbnailUrl": "缩略图地址"
    }
  ]
}
```

## 下一步建议

1. 做一个小型 115 API 转接服务，负责授权、刷新令牌、读取文件列表。
2. 增加缩略图缓存，避免每次重新加载。
3. 增加标签持久化，把收藏、稍后整理保存到本地数据库。
4. 做成桌面应用，比如 Tauri 或 Electron。
5. 加入自然语言搜索，比如“找去年旅行的视频”。
